Busquedas Avancadas de manera fácil
Se pueda aumentar la exactituda de las busquedas si se agregan operadores que afine las palabras y números. Todas las opciones enumeradas en esta página se captura directamente en la caja de busqueda.
Basicos
Se puede capturar una palabra/número o una expresión, ejem. 'Miller', '> 500' or '>= Smith', y enseguida
• dar click en el encabezado de la columna para buscar objetos a los que sus valores correspondan con un atributo en particular (Se buscará en todos los objetos sin importar si se tienen filtros configurados o si hay alguna busqueda actual cuyos resultados se perderan) Busqueda Nueva
• dar click en el [+] boton para buscar los objetos con valores que correspondan con un atributo en particular (solo se buscará en los objetos contenidos en la busqueda o filtro previos , ejem. 'agregar' esta busqueda regresara un conjunto de resutlados mas restringido) Agregar Condición
Operadores
Los siguientes operadores están soportados (notese, sinembargo, que algunos sistemas de base de datos no soportan todos los operadores):
  • sounds like (*)
  • like (%), operador default para busquedas de texto
  • equal (=), operador defaulta para busquedas booleanas, de números y de códigos
  • not equal (<>, !=)
  • less (<)
  • less or equal (<=)
  • greater (>)
  • greater or equal (>=)
"AND" Busquedas
Para traer objetos que corresponden con una o mas condiciones al mismo tiempo, usese el amperson "&" entre las condiciones. Por ejemplo, para buscar los objetos con valores entre 500 y 1000 (ejem. mayores que 500 y menores que 1000), teclear: '>500 & <1000'
"OR" Busquedas
Para traer los objetos que inlcuyen ya sea la palabra A o la palabra B, usar el "punto y coma" ";" entre las condiciones. Por ejemplo, para buscar los contactos llamados 'Smith' ó 'Miller', teclear: '= Smith; Miller'
Busquedas Combinadas
Se pueden combinar las busquedas de manera practicamente ilimitada. Por ejemplo, para buscar los objetos con valores menores a '1000', pero diferentes de '500' y diferentes de '600', teclear: '<>500; 600 & <1000'
Las reglas de análisis de busquedas combinadas son como sigue:
  • searchExpr ::= filterValue { & filterValue }
    (--> string tokenizer with & as separator)
  • filterValue ::= [operator] value { ; value }
    (--> string tokenizer with ; as separator)
Busquedas con Comodines

Algunas Bases de Datos permiten busquedas con comodines. Si estás se soportan se pueden buscar todos los contactos con nombres que contienen s containing 'ller' (ejem. Miller, Mueller, etc.) tecleando: '%%ller'
ADVERTENCIA: las busquedas con comodines pueden consimir bastante tiempo si la base de datos no esta afinadas para eso. (busquedas completas en la tabla)!
